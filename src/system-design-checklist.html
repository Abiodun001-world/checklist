Week 1: Fundamentals and System Components

  • Idea behind load balancer
    Load balancing is the process of distributing a set of tasks over a set of resources. Different types of load balancing algorithms are used to distribute incoming network traffic across multiple servers, ensuring optimal resource utilization and improved performance.
  • Types of load balancing algorithms
    Load balancing algorithms are crucial for distributing incoming network traffic across multiple servers, thats why file uploading service, rate limiting systems and URL shortener uses different algos.
  • System design of file uploading service
    System design of file uploading service involves designing a scalable and reliable system to handle file uploads. Load balancing, caching and data partitioning are important factors of this topic.
  • Idea of vertical and horizontal scaling
    Vertical scaling involves increasing the resources of a single server or node, such as CPU, memory, or storage capacity, to handle greater workloads. It is typically limited by the hardware's capacity and can result in higher costs.
  • Database replication
    Database replication is the process of creating and maintaining copies of a database to enhance availability, reliability, and performance. Database replication is crucial for: implementing fault tolerance , consistency pattern, etc.
  • Data partitioning in system design
    Data partitioning, also known as sharding, is the practice of dividing a large dataset across multiple storage nodes or database instances. It enables horizontal scaling and improves performance and availability.
  • Consistent hashing
    Consistent hashing is a distributed hashing technique that provides scalability and load balancing in distributed systems. You need to have knowledge in load balancing algorithms, distrubuted systems and data partitioning.
  • Sidecar design pattern in system design
    Sidecar design pattern in system design is a technique where a separate container or process runs alongside a primary application, providing additional functionality or services. It promotes modularity and flexibility in system architecture. Some relevent topics to explore are containerization, microservices and service discovery

Week 2: Data Storage and Caching

  • Different types of databases
    In the world of system design, databases play a crucial role in storing and managing data efficiently. Understanding the different types of databases is essential for designing scalable and robust systems. Here are some important topics to explore: ER diagram/entity relationship model, Distributed Database Management System (Distributed DBMS), System design of distributed cache, System Design of Instagram.
  • Choose the right type of NoSQL database
    When selecting a NoSQL database, several factors come into play, including scalability, data model, consistency requirements, and query patterns. Some topics to explore: Consistency patterns in System Design, Data Partitioning in System Design, Database replication.
  • System design of a URL shortner
    System design of a URL shortener is a process that involves designing a service to shorten long URLs while maintaining their accessibility and redirect functionality. Some important topics to understand for this include: Idea behind load balancer Types of load balancing algorithms, Different types of databases, Caching stratergies.
  • In-memory database
    An in-memory database stores data in the main memory (RAM) for faster access and lower latency compared to traditional disk-based databases. Some topics to explore are: System design of file uploading service, Different types of databases.
  • Types of caching in web applications
    This is an important topic as caching plays a vital role in improving the performance and reducing the load on backend systems. In web applications, different types of caching techniques are employed. Some topics to explore are: Caching Strategies in system design
  • Different caching strategies in system design
    Different caching strategies in system design are crucial for improving performance and reducing the load on backend systems. Here are a few important topics related to caching that you can explore further: Distributed cache, Cache Stampede, MemCached in system design.
  • System design of a firewall
    A firewall is a crucial component for network security. It involves: Layer 4 and layer 7 load balancing, Stateful and stateless architecture, Security and authentication.
  • System design of GitLab
    GitLab is a popular DevOps platform. Its system design encompasses aspects such as database replication, distributed systems, containerization, and load balancing algorithms. Some topics to explore: Distributed systems, Database replication, Containerization, Load balancing algorithms.
